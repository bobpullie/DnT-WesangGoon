## 3. 게임 심리학 핵심 원칙 (Game Psychology Core Principle)

**게임/인터랙티브 시스템의 모든 설계·평가·빌드에서 인간 심리와 감정 메커니즘을 핵심 설계 원칙으로 적용한다.**

기술 구현이나 기능 목록 이전에 반드시 유저 심리 모델을 먼저 정의할 것:
1. **타겟 감정** — 이 기능이 유발해야 할 감정은 무엇인가
2. **심리 메커니즘** — 어떤 인지 편향/보상 회로를 활용하는가
   - 손실 회피, 앵커링, 변동 비율 강화, 도파민 루프, 기대심리, 사회적 증거, Flow state 등
3. **행동 유도** — 유저의 어떤 행동을 유발/억제하려는가
4. **중독성 vs 건전성 경계** — 사행성/과몰입 방지선은 어디인가

이 4가지가 명시되지 않은 게임 설계는 불완전하다.

## 4. TEMS 접근 (Topological Evolving Memory System)

### 4.1 메모리 DB 위치
- `E:\AgentInterface\memory\error_logs.db` — 위상군 마스터 DB
- 프로젝트별 TEMS DB가 존재할 수 있음 (예: `E:\MRV\tems\tems_db.db`)

### 4.2 작업 전 자가 검색 (Preflight)
새로운 코드를 작성하거나 에러를 마주했을 때, 즉시 해결하지 말고:
```python
from memory.fts5_memory import MemoryDB
db = MemoryDB()
results = db.preflight("관련 키워드")
```
과거 교훈을 먼저 확인하고, 결과에 따라 접근 방식을 선언한 후 진행.

### 4.3 교훈 기록 (Commit)
작업 중 새로운 교훈(실패/성공)을 발견하면 즉시 기록:
- TCL: 사용자 지시 기반 체크리스트 → `db.commit_tcl(...)`
- TGL: 실수/시행착오 기반 가드 → `db.commit_tgl(...)`
