# 아키텍처 제약 & 엔트로피 관리

## 결정론적 검증 (Deterministic Validation)
- 위상수학적 공식이나 텐서 차원 축소 코드는 엄격한 정적 타입 힌팅 사용
- 코드 작성 후 가상의 `assert`로 차원(Dimension) 매핑 자가 검증

## 가비지 컬렉션 (엔트로피 억제)
- 긴 코딩 세션 후 문서-코드 불일치('컨텍스트 부패') 점검
- 미사용 코드 제거, 문서 일관성 점검

## 지식 한계 인식 (No Hallucination)
- 불확실한 영역에서 추측하지 말 것
- **"이 부분은 현재 컨텍스트에 부족합니다."** 선언 후 WebSearch/WebFetch로 조사
- 도메인 기획(게임 메커니즘, UX, 심리학)은 기획군에 위임

## 설계 자문 프로토콜 (Design Self-Check)
나선형 시스템 설계 시 매 판단마다 자문:
1. "이 기능은 나선의 **어떤 단계**를 지원하는가?"
2. "여기서 인간의 역할은 **수렴(결정)**인가, AI의 역할은 **발산(탐색)**인가?"
3. "이 결정의 위험도는? **라우팅**은 적절한가?"
4. "이전 반복의 교훈이 **TEMS를 통해** 이번 설계에 반영되고 있는가?"
