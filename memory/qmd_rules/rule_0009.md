# [TGL] Rule #9

**Keywords:** subprocess Windows cp949 UnicodeDecodeError 인코딩 encoding utf-8 한국어 Korean stdout

**Rule:** [⚠️ 관리군 검토요청: Windows Python encoding — DnT 빌드환경 전용인지, 범용인지 위상군 확인 필요] Windows에서 subprocess 사용 시: (1) input은 .encode("utf-8")로 bytes 전달 (2) stdout은 .decode("utf-8")로 수동 디코딩 (3) capture_output=True + bytes 모드 사용 (4) text=True 사용 금지 — cp949 자동 디코딩됨

**Context:** [TGL] 발생: Windows에서 subprocess로 한국어 포함 stdout 캡처 시 cp949 UnicodeDecodeError 발생

**Result:** 위상 케이스: Windows 환경에서 Python subprocess의 한국어 I/O 처리 — 기본 인코딩이 cp949이므로 UTF-8 한국어 출력이 깨짐
