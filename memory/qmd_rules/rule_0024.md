# [TCL] Rule #24

**Keywords:** 규모 스케일 확장 커지 파이프라인 컨텍스트 기억 손실 구멍 빌드 대규모 프로젝트 관리

**Rule:** 프로젝트 규모가 커지면 에이전트의 컨텍스트 윈도우 한계로 기억 손실과 구멍이 발생. TEMS는 규칙/교훈 보존이지 작업 맥락 보존이 아님. 기획/설계/빌드 모든 단계에서 규모 확장에 대응하는 업무 분할, 코드베이스 스코핑, 점진적 컨텍스트 로딩 파이프라인 설계가 필요.

**Context:** 종일군이 프로젝트 규모 확장 시 에이전트 업무대응 파이프라인 설계를 요청

**Result:** 다음 세션 최우선 TODO로 등록
