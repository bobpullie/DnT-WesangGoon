# [TGL] Rule #37

**Keywords:** useEffect deps snapshot 캡처 시점고정 klineCount submitIndex 동적값

**Rule:** useEffect가 특정 시점의 값을 캡처하는 용도라면 캡처 대상을 deps에서 제외하고 트리거만 deps에 넣을 것. eslint exhaustive-deps 경고는 이 경우 의도적 억제.

**Context:** [TGL] 발생: TimeHighlight의 useEffect deps에 klineCount를 포함하여 새 캔들마다 submitIndex가 최신으로 갱신됨. prediction window가 시간축에 고정되지 않고 항상 최신 캔들 앞에 위치.

**Result:** 위상 케이스: useEffect가 특정 시점의 스냅샷을 캡처하는 용도인데, deps에 동적 변화 값을 포함하면 캡처가 매번 갱신되어 고정이 안 됨.
