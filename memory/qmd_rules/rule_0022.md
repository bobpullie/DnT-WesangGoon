# [TCL] Rule #22

**Keywords:** 요청 제안 의견 방법 구현 설계 구상 아이디어 하고싶 만들고싶 이렇게 해볼까 어떨까

**Rule:** /meta-check 스킬로 절차적 구현됨. 원본: 종일군 요청 → 의지/방법 메타 추출 → 최적성 검증

**Context:** 종일군의 모든 요청에 의지(1)와 방법(2) 메타 분리 분석 시작

**Result:** 방법(2)이 의지(1)에 부합하지 않는 경우 발견하여 더 나은 대안 제시 성공
