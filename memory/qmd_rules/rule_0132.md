# [TGL] Rule #132

**Keywords:** 문제없 깨끗 완료 clean 통과 passing 모순 자기모순 자가감사 self-audit 일관성 단언 단정 발화 직전

**Rule:** [TGL-C / L2] 직전 발화의 단언과 직후 발화의 부채 나열이 같은 응답 내에서 충돌하면 사용자에게 신뢰 손상 + 검증 절차 누락의 표지. 발화 일관성은 결정론 게이트(DVC/grep/test) 통과 후에만 가능. self_cognition_gate 가 'numeric_self_audit_falsification' 분류로 비슷한 케이스를 매칭하지만 false positive 빈발 — 본 TGL-C 가 정식 일반화. S55 본 세션 직접 위반 사례: 'PC/사용자 문제없습니다' 단언 후 같은 응답에 4건 하드코딩 부채 나열.
FORBIDDEN: 결정론 검증 첨부 없이 '문제없' / '완료' / 'clean' 단언을 한 후 같은 응답에 부채/결함/위반 N건 나열.
REQUIRED: 단언 전에 부채 카운트 0 증명 (DVC case 출력 / grep 결과 / test PASS) 첨부. 부채 N>0 이면 단언 자체를 '부채 N건 잔존 — 단계 진행 여부 판단 요' 로 교체.
