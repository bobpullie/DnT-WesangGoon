# [TCL] Rule #135

**Keywords:** 플러그인 plugin 스킬 skill 원격 레포 repo remote github upstream 업데이트 update git pull push 클론 clone TEMS TWK SDC DVC handover bobpullie canonical 5-asset 5-Asset Five-Asset 자산 4-원칙 four-principle independence local-only separate-repo universal portability 범용성 독립성 로컬한정 hub 동기화 sync frontmatter SKILL.md update_cmd 기능업데이트 cross-agent 에이전트간 push누락 미동기화 drift

**Rule:** 5개 플러그인/스킬 자체 자산 체계 (TEMS / SDC / DVC / TWK / handover) — 각자 GitHub canonical 레포: TEMS=https://github.com/bobpullie/TEMS, SDC=https://github.com/bobpullie/SDC, DVC=https://github.com/bobpullie/DVC, TWK=https://github.com/bobpullie/TWK, handover=https://github.com/bobpullie/handover. 4 push 게이트 원칙(모두 통과해야 canonical push 허용): (1) Independence — 5자산 상호 의존 0, 각각 self-contained. (2) Local-Only — 모든 에이전트 자산(룰/DB/QMD/핸드오버/위키)은 에이전트 로컬 프로젝트 폴더 안에만, hub 디렉토리 금지. (3) Separate-Repo — 각자 별도 canonical 레포, 한 PR 에 두 레포 묶기 금지. (4) Universal Portability — Windows/Linux/macOS, 임의 OS user, 임의 에이전트명에서 작동 — 절대경로/특정 user-name/hub 의존 금지, env var · marker walk · Path(__file__) · importlib.resources 만 허용. 위상군 또는 다른 에이전트가 기능 업데이트 시 해당 canonical 레포에 commit+push → 전체 에이전트는 git pull / pip install -U / curl 로 수신. 로컬 수정만 후 canonical push 누락 금지 — 다음 update_cmd 시 변경분 소멸. SKILL.md frontmatter 의 upstream+update_cmd 필드 필수.
