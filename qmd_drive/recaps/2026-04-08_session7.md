# QMD — 2026-04-08 Session 7

## 주제
KH Phase 1 수동 검증 부트스트랩 — Obsidian 설치, 노트 워크플로우 구축, 그래프 시각화 메커니즘.

## 교훈 (TEMS 후보)
1. **Obsidian Graph View는 `[[wikilink]]`만 엣지로 인식.** frontmatter 평문 필드(`project: DnT_Game`)는 그래프에 반영되지 않음. 그래프 클러스터링이 필요한 vault는 처음부터 link 형식 frontmatter 또는 본문 link 라인 컨벤션을 정해야 함. → TGL 후보.
2. **Dataview 스키마와 그래프 엣지 메커니즘이 충돌**할 때, 검증 단계 비용을 우선해 본문 link 라인으로 우회 → 정식 통합은 다음 phase로 미루는 패턴. (TDD의 "make it work, then refactor" 변주) → TCL 후보.
3. **하드코딩 회피 = 동적 스캔 + 컨벤션 도입**. Templater의 `app.vault.getMarkdownFiles()` + `_` prefix 메타파일 컨벤션으로 새 항목 추가 시 단일 변경점 확보.
4. **Templater suggester/prompt vs 코어 Templates**: 드롭다운/대화형 입력은 Templater 전용. 코어 Templates는 정적 치환만 가능. 설계 시점에 결정해야 함.
5. **종일군이 직접 검증하는 단계에서는 환경 설치(Obsidian)부터 막힐 수 있음.** 위상군이 winget으로 직접 설치 처리. "검증 단계 = 코드만 확인"이 아니라 "종일군 환경에서 end-to-end 작동"이 진짜 검증.
