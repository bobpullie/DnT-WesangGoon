## 핵심 결정
1. **viewer main guard 의 `__mp_main__` 분기 제거** — S48 의 "native 모드 실패 → --browser fallback" 결정은 진단 불완전. 실제 원인은 NiceGUI native_mode 의 mp.Process 자식 spawn 시 `__mp_main__` 가드가 main() 재진입을 유발. 1 토큰 fix 로 native 모드 정상화
2. **launcher = pythonw.exe + CREATE_NO_WINDOW** — python.exe console subsystem 이 conhost.exe 깜빡임 유발. Windows 분기에서 pythonw.exe 우선 사용 + CREATE_NO_WINDOW 플래그로 자식 트리에서 conhost.exe 0건 보장
3. **layout fix 는 chain 전체 강제** — Quasar default `display: block` 인 q-layout / q-page-container 가 자식의 height:100% / flex:1 1 auto 를 0 또는 자연 높이로 해석 → 무한 grow → body overflow:hidden 으로 잘림. chain 전체 (`html, body, body > div, .nicegui-layout, .q-page-container, .q-page-container > .q-page, .q-page > .nicegui-content`) 강제 `display: flex !important; flex-direction: column !important; flex: 1 1 auto !important; min-height: 0 !important;` + 최상단 100vh + `.q-table__middle { overflow: auto }` + 형제 sticky `flex: 0 0 auto`
4. **canonical 분리** = bobpullie/TEMS_GUI_Editor (viewer 전용) + 위상군 로컬 viewer/ snapshot. bobpullie/TEMS / TWK / handover 와 같은 패턴
