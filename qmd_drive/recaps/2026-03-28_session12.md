# Session 12 — 과녁 배치 UX v2.0 (모바일 퍼스트) + ADR-002-R1/ADR-010

## 핵심 성과
1. 과녁 배치 UX v1.0 (PC 기반 탭) 기획 → 종일군 피드백으로 기각
2. 과녁 배치 UX v2.0 (모바일 퍼스트, 아이템 슬롯→드래그) 정식 채택
3. ADR-002-R1 작성 (sharpness 제거, ATR 거리 기반 채점)
4. ADR-010 작성 (아이템 슬롯 + Origin-Based Touch Routing + UX 4 Layer)
5. 궤적 통합 방향 확립 (짧은 모드: 슬롯 드래그, 긴 모드: 궤적 끝점=과녁)

## 핵심 교훈
- 기획군에 플랫폼 컨텍스트(모바일 퍼스트)를 전달하지 않으면 PC 기준 설계가 나옴 — 기획 요청 시 타겟 플랫폼 명시 필수
- Origin-Based Routing(터치 시작 영역으로 이벤트 결정)은 모바일 터치 이벤트 충돌의 구조적 해결책
- 종일군의 게임적 비전(마인크래프트 슬롯)이 UX 문제를 구조적으로 해결 — 디렉터의 게임 직감을 기술적으로 검증하는 것이 위상군의 역할

## 다음 세션 방향
- 커밋+푸시 → 밴드→과녁 빌드 (ADR-002-R1 + ADR-010) → Turn 1 TEST/POSTMORTEM
