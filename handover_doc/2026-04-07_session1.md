Corvis (OpenClaw) 에이전트 인프라 대폭 강화 세션. Web UI 접속, 도구 프로비저닝, 스킬 확장, MCP 서버 연동까지 완료.

## 주요 작업

### 1. Corvis Web UI 접속 활성화
- **문제:** `ws://127.0.0.1:18789`는 Docker 내부 loopback 바인딩이라 브라우저 접속 불가
- **해결:** `openclaw.json`에 `"bind": "0.0.0.0"` 추가 (OpenClaw이 `"lan"`으로 자동 정규화)
- **페어링:** 외부 연결은 device pairing 필요 → `openclaw devices approve` 명령으로 승인
- **토큰 고정:** `gateway.auth.token` 고정값 설정 (TGL #62 대응)

### 2. Docker 도구 자동 프로비저닝
- `docker-compose.yml` 대폭 수정:
  - `user: root`로 전환 → apt 설치 가능하게
  - entrypoint에서 apt + npm 도구 자동 설치
  - `openclaw-tools` 볼륨: 프로비저닝 마커 영구 보존
  - `openclaw-state` 볼륨: 페어링/세션/device 데이터 영구 보존
  - 설치 후 `su node`로 전환하여 gateway 실행
- **설치된 apt 패키지:** jq, ripgrep, tmux, ffmpeg, htop, tree, unzip, less, procps, gh (GitHub CLI), python3-pip, python3-venv
- **설치된 npm 패키지:** @google/gemini-cli, clawhub, summarize
- **주의:** apt 패키지는 컨테이너 recreate 시 재설치 필요 (entrypoint에 재설치 로직 포함)

### 3. 스킬 확장 (6개 → 17개)

- 도구 설치로 추가 (5개): github, gh-issues, session-logs, tmux, video-frames
- npm 설치로 추가 (3개): gemini, clawhub, summarize

### 4. Kimi K2.5 Primary 모델 전환

- `openclaw.json` primary: `moonshot/kimi-k2.5`, fallbacks: `google/gemini-2.5-flash`, `ollama/gemma4:e4b`
- `docker-compose.yml`에 `MOONSHOT_API_KEY` 환경변수 추가

### 5. ComfyUI MCP 서버 연동
- `joenorton/comfyui-mcp-server` 클론 → `E:\02_openclawAgent\comfyui-mcp-server\`
- `server.py` 수정:
  - `sys.exit(1)` 제거 → ComfyUI 없이도 MCP 서버 기동 가능
  - `host="0.0.0.0"` 추가 → Docker에서 `host.docker.internal:9000` 접근 가능
- `mcpServers.json`에 comfyui 엔트리 추가
- `start-comfyui-mcp.bat` + Windows 시작프로그램 바로가기 생성

### 6. GOOGLE_API_KEY 무료등급 키 재발생 문제
- Process 레벨 환경변수에 무료등급 키(`AIzaSyAoqFkA...`)가 남아있어 `.env`의 결제활성화 키를 덮어씀
- `unset GOOGLE_API_KEY && docker compose up -d`로 해결
- 근본 원인: Docker Desktop이나 셸 세션 시작 시 상속된 환경변수 → 컴퓨터 재시작하면 해소 예상
