# 위상군 세션 핸드오버 — 2026-04-06 Session 5

## 세션 목적
Corvis(OpenClaw) 에이전트에 Kimi K2.5 모델 추가 및 Gemini API 429 에러 해결

## 수행 작업

### 1. Kimi K2.5 (NVIDIA NIM) 모델 추가 시도
- **provider:** nvidia, baseUrl: `https://integrate.api.nvidia.com/v1`
- **model ID:** `moonshotai/kimi-k2.5`, api: `openai-completions`
- **결과:** 연결 성공, 응답은 오지만 **1~2분 지연** (NVIDIA NIM 무료 API cold start + 공유 GPU)
- **결정:** 너무 느려서 보류, 전체 변경사항 git rollback (`git checkout --`)

### 2. 설정 과정에서 발견한 이슈들
- `openai-chat` → 유효하지 않은 API 타입. 유효 목록: `openai-completions`, `openai-responses`, `anthropic-messages`, `google-generative-ai`, `ollama` 등
- `openai-responses` → NVIDIA NIM에 `/v1/responses` 엔드포인트 없음 (404)
- `openai-completions` → 정상 (NVIDIA NIM `/v1/chat/completions` 매핑)
- docker-compose.yml `environment`에 `NVIDIA_API_KEY` 명시적 추가 필요했음
- gateway auth token이 `down/up` 시 재생성되어 TUI 인증 실패 → `openclaw.json`에 token 고정으로 해결

### 3. Gemini 429 에러 근본 원인 발견 및 해결
- **증상:** `generate_content_free_tier_requests` limit: 20 (무료등급)
- **원인:** Windows User 환경변수에 이전(무료등급) `GOOGLE_API_KEY`가 등록되어 `.env`의 결제활성화 키를 덮어쓰고 있었음
  - `.env`: `AIzaSyBNNsz8tqv...EIjAY` (결제활성화, RPD 500)
  - User env var → 컨테이너: `AIzaSyAoqFkAhr...Wk6J8` (무료등급, RPD 20)
- **해결:** `[Environment]::SetEnvironmentVariable('GOOGLE_API_KEY', $null, 'User')` 로 User 환경변수 삭제
- **결과:** `.env`의 결제활성화 키가 정상 적용, 429 에러 해소
