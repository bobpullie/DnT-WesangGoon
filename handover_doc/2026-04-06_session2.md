### 블로커 2: Ollama delivery-mirror 타임아웃
- **증상:** 텔레그램 → OpenClaw agent → Ollama 경로에서 120초 후 타임아웃
- **로그:** `provider=openclaw/delivery-mirror reason=timeout` → `candidate=ollama/gemma4:e4b reason=rate_limit` (실제로는 타임아웃의 오분류)
- **원인 추정:** OpenClaw의 delivery-mirror 프록시가 로컬 Ollama의 OpenAI 호환 API를 제대로 처리하지 못함. native `api: "ollama"` 타입도 동일 증상
- **작동하는 것:** curl 직접 호출, `openclaw agent --local` → 정상
- **작동 안 하는 것:** 텔레그램 → delivery-mirror → Ollama
- **시도한 것:**
  1. `api: "ollama"` (native) → 타임아웃
  2. `api: "openai-completions"` + `/v1` baseUrl → 타임아웃 (모델 없을 때는 404, 모델 있을 때는 타임아웃)
  3. billingBackoffHours 최소화 → 효과 없음
  4. idleTimeoutSeconds 120 → 타임아웃이 60→120초로 늘어났을 뿐
- **다음 조사 방향:**
  1. OpenClaw docs에서 delivery-mirror + 로컬 모델 설정 방법 확인
  2. `openclaw doctor` 명령으로 진단
  3. OpenClaw Discord/GitHub에서 유사 이슈 검색
  4. delivery-mirror 우회: 텔레그램 대신 WebSocket/API 직접 호출 테스트
