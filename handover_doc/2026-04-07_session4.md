## 주요 설계 결정 / 어댑테이션

1. **Connection lifecycle 패턴 통일 (Task 13 리뷰어 제안)**
   - `with self._conn() as conn` NOT 사용 — sqlite3 context manager는 트랜잭션만 관리, 연결 close 안 함
   - 전체 WikiAPI를 `conn=self._conn(); try: ...; finally: conn.close()` 로 통일
   - Task 14도 동일 패턴 유지

2. **Task 14 name shadowing 회피**
   - WikiAPI 인스턴스 메서드 `get_node`/`update_node`가 모듈 top의 동명 함수를 가림
   - 해결: 모듈 import alias `from .nodes import get_node as _get_node, update_node as _update_node`
   - Task 13이 이미 작성한 기존 import도 alias에 맞게 수정됨

3. **`get_pending_tasks` NULL 처리**
   - `status != 'done'`는 SQL에서 NULL 제외 → 테스트 fixture task-1 (status="pending") 누락 위험 예방
   - `(status IS NULL OR status != 'done')` 으로 교정

4. **Task 9 dead branch (리뷰어 minor)**
   - `node_type = "session"` 양 arm 동일 → Phase 1 스코프상 harmless tautology, 단순화 차후

5. **Task 10 orphan node 우려 (리뷰어 minor, Phase 3 backlog)**
   - 해시 변경 시 `generate_node_id` 재호출로 기존 mirrors row의 `node_id`가 무효화 가능
   - 현재는 hash-skip 가드로 방어, 향후 update path 별도 필요
