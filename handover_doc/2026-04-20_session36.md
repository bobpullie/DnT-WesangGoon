### 1. TWK Dataview+Calendar 호환화 (요청 #1)

- 종일군 질문: TWK 의 frontmatter 양식이 Dataview 플러그인 포맷을 따르는가? Calendar 의 날짜 포맷과 일치시켜 연동 가능한가?
- 조사 결과 (general-purpose subagent 위임):
  - Dataview 는 field 스키마리스. `date: YYYY-MM-DD` 무인용은 Date 타입으로 자동 coerce; 인용 시 Text 로 전락하여 `WHERE date > date(today)` 비교 깨짐.
  - Calendar 는 frontmatter 를 읽지 **않음**. Daily Notes 코어 플러그인의 filename format (기본 `YYYY-MM-DD`) 만으로 dot 을 그림.
  - 통합 해법: daily note 를 **허브**로 두고 `WHERE date = this.file.day` Dataview 쿼리로 같은 날짜 wiki 페이지 자동 역참조.
- 기존 TWK 템플릿 감사 결과 **6개 중 5개가 frontmatter 완전 부재** (concept/entity/idea/postmortem/principle). decision.md 만 있음. 이 상태로 copy 시 lint 실패 구조적 결함.

### 5. SDC Gate Phase 3 편입 — 실시간 violation fix (`8b5cc06`)

- **트리거 사건**: 세션 중 종일군이 "그럼 이건 규칙 위반 아닌가?" 지적. 확인 결과 commit+push 그룹 분할 작업에서 본체가 SDC 3-question gate verbalize 생략 → TCL #120 위반.
- **Root cause**: SDC 는 self-dispatch 규칙이라 자동 강제 없음. 조용히 생략 가능한 구조적 취약점.
- **해결 결정** (Q3 Reversibility = Local but critical-infra → DELEGATE + STAGING):
  - Sonnet general-purpose 에 brief 5항목 위임 (`Agent` tool, model="sonnet").
  - Sonnet: `check_sdc_gate()` 함수 작성 + main() 통합 + 5 smoke tests + `git add` 까지만.
  - Opus 본체 trust-but-verify: pytest 재실행 (5/5 pass), diff 검토, 기존 TGL-T 회귀 0 확인.
  - Opus 본체가 commit + push 수행 (TCL #86 STAGING 프로토콜).
- **구현 상세**:
  - regex: `\bgit\s+(commit|push|merge|rebase|cherry-pick|revert)\b`
  - `SELF_INVOCATION_MARKERS` 재사용 (TEMS 자체 호출 루프 방지)
  - `active_guards.json.sdc_brief_submitted == True` 면 gate clear
  - warning only (deny 승격 금지)
  - stdout 에 `<sdc-gate-alert>` 태그 append — 본체 컨텍스트 주입
- **오버헤드**: p50 +5~10ms, 세션당 ~$0.005 context tokens, 개발 공수 1 세션. 위반 1건 교정 비용($1~2) 대비 200:1 효율.
