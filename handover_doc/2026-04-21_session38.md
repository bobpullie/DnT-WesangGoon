**문제 인식:** 기존 SDC.md §0는 매 task 수신 시 Auto-Trigger 키워드(commit/push/migrate/이식/…)를 LLM이 자의적으로 탐색. 일반 대화나 설계 질문에도 §0 진입 비용 발생. rule-based로 전환하면 TEMS preflight의 selective 매칭 메커니즘에 얹을 수 있음.

**설계 (brainstorming 4문답):**

  - **rule-based (기본)**: `tags: sdc_trigger` TCL이 task에 매칭돼 preflight 주입 시에만 3-question gate 수행

### 2. TCL #122 등록 — git 배포 전용 SDC 트리거

```bash
python memory/tems_commit.py --type TCL \
  --rule "앞으로 git commit/push/merge/rebase/cherry-pick 등 배포 명령 수행 전 SDC 3-question gate 수행 (Q1 Invariance / Q2 Overhead / Q3 Reversibility). Shared state 수정이므로 KEEP/DELEGATE+STAGING 판정 명시" \
  --triggers "git, commit, push, merge, rebase, cherry-pick, deploy, 배포, 푸시, 커밋, 병합" \
  --tags "sdc_trigger,project:meta"
```

A2 키워드 다양성 경고 발생(slash 구분자 파싱 오차)이지만 실전 매칭은 정상. 등록 직후 `"git push origin master 해줘"` prompt에서 `#122 [SDC]` 마커 주입 확인.

코드 구현 관련 TCL은 의도적 미등록 — triggers 범위가 넓어 rule-based의 selective 장점이 희석될 우려. 세션 경과 관찰 후 필요 시 추가 판단.
