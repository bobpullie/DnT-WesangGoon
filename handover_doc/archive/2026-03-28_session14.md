### 3. 종합 UI/UX 스캐폴딩

종일군 테스트 후 7건 피드백:
1. 아이템슬롯 차트 중앙 배치
2. 유저 정보(닉네임, 랭킹, 레벨, 스코어 등) 고정 표시 공간 부재
3. Auto-Stake Long/Short 인식 불가 (너무 작음)
4. RPG식 데미지/힐 연출 필요 (+21, -10)
5. 게임 문법 준수 (게임 UI)
6. 모바일 해상도 최적화
7. 폰트 크기 전반적으로 작음

기획군 진단: **"현재 DnT는 게임이 아니라 차트 앱"** — 유저가 게임 속 자기 자신을 인식할 방법이 없음

### 4. 화면 방향 + 해상도 확정

#### 가로형 확정 (종일군 + 외부 자문)
- 차트 시간축이 가로 → 캔들 20-30개 표시 (세로면 8개)
- 집중 플레이 세션 → 가로 회전 마찰 무시 가능
- 레퍼런스: 트레이딩앱, 4X전략, 타워디펜스 모두 가로

#### 해상도 확정 (종일군 + 외부 자문)
- **디자인 기준 해상도: 1280 x 720** (논리 픽셀)
- CSS viewport: 640-840 x 360-420 (기기별 가변, 디자인 기준의 0.5x~0.65x)
- 차트 렌더: DPR 네이티브 (lightweight-charts 자동 처리)
- lightweight-charts는 이미 DPR을 내장 처리하므로 차트 선명도 문제 없음

**기획서:** `docs/planning/DnT_M2_Phase0_Design_1280x720.md`

| NAV  | HUD (45px)                             | SIDE |

- 차트 리사이즈 제로 (상태 전이 시 크기 불변)
- P&L 이중 표시: 사이드(상세) + 차트 좌상단(축약)
