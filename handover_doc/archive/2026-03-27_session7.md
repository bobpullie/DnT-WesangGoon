### 2.5 레이아웃 버그 수정

**종일군 보고:** "첫 접속 시 레이아웃 전체가 왼쪽으로 쏠려서 좌측 메인 메뉴가 안 보임. 한 턴 후 정상화."

**수정:**

- `CandlestickChart.tsx` — `createChart` 후 `rAF`로 강제 resize 호출

### 2.6 LVS (Layout Validation System) 구축

**종일군 요청:** "시각적 요소를 에이전트가 자동으로 검증하는 체계"

Galen Framework의 선언적 제약 조건 패턴을 React 런타임 훅으로 구현:

**파일:**
- `dev/layoutValidator.ts` — 12종 룰 엔진 + `GAME_LAYOUT_SPECS` (4상태)
- `dev/useLayoutValidator.ts` — React 훅: 제약 조건 검증 + PerformanceObserver LayoutShift 감지
- `dev/index.ts` — exports

**12종 룰:** `visible` | `hidden` | `insideViewport` | `width` | `height` | `minWidth` | `minHeight` | `leftEdge` | `noOverlap` | `leftOf` | `rightOf` | `contains`

**에이전트 접근점:**
- `window.__layoutReport` — 최신 검증 리포트
- `window.__layoutReportHistory` — 최근 20건 히스토리
- `window.__layoutShifts` — LayoutShift API 감지 이력

**통합:** GameScreen + NavRail에 `data-layout` 속성 태깅 (header, chartArea, sidePanel, bottomBar, navRail)

## 6. 위상군 자가 반성

### NavRail transition 버그의 교훈
Session 7에서 직접 작성한 CSS transition이 초기 마운트에서 layout shift를 유발했다. CSS transition이 있는 요소를 새로 마운트할 때 **초기 렌더에서 transition을 비활성화하고 rAF 후 활성화**하는 패턴을 TGL로 커밋. 이 패턴은 향후 모든 transition 요소에 적용해야 한다.

### LVS 체계의 의의
종일군이 "시각적 요소를 에이전트가 검증하는 체계"를 요청. 이것은 에이전트 팀의 근본적 한계(화면을 볼 수 없음)에 대한 체계적 해결책이다. Galen Framework 패턴을 React 런타임 훅으로 구현하여, 빌드 후 상태별 레이아웃 제약 조건을 자동 검증하고, 에이전트가 `window.__layoutReport`로 구조화된 결과를 읽을 수 있도록 했다. 앞으로 새 화면/상태 추가 시 `GAME_LAYOUT_SPECS`에 스펙을 함께 정의하는 것이 필수.

### Thin Router 효과 지속
이번 세션에서 기획군 2회, 리서치 에이전트 1회 가동하면서 컨텍스트 상한에 도달하지 않음.
