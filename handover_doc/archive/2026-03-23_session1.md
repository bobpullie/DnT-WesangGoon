## 2. 완료된 작업 (상세)

### 2.1 FTS5+BM25 메모리 시스템 구축
- `memory/fts5_memory.py` — MemoryDB 클래스 생성
- `memory/error_logs.db` — SQLite FTS5 기반 DB 생성
- 스키마: memory_logs 테이블 + memory_fts 가상 테이블 + 자동 동기화 트리거(INSERT/UPDATE/DELETE)
- `keyword_trigger` 컬럼 추가 (v2 확장)
- `summary` 컬럼 추가 (컨텍스트 압축용)
- `commit_memory()`, `search()`, `preflight()`, `commit_tcl()`, `commit_tgl()` 메서드 구현

### 2.2 TCL/TGL 시스템
- **TCL (Topological Checklist Loop):** 사용자의 "앞으로" 지시를 위상적 체크리스트로 변환
- **TGL (Topological Guard Loop):** 실수/시행착오를 위상적 가드 규칙으로 변환
- 등록된 TCL 규칙 8건, TGL 규칙 1건 (Windows cp949 인코딩)
- DB 총 레코드: 13건

### 2.3 UserPromptSubmit Hook (하드 트리거)
- `memory/preflight_hook.py` — 매 프롬프트마다 자동 실행
- 한국어 어미 분리(`strip_korean_suffix`) + prefix BM25 검색
- `.claude/settings.local.json`에 hook 등록 완료
- 7/7 테스트 통과 확인

### 2.4 TEMS 아키텍처 (4 Phase)
- `memory/tems_engine.py` — 전체 TEMS 엔진
- **Phase 1:** HybridRetriever — FTS5 BM25 + QMD 벡터 검색 + Dynamic RRF
- **Phase 2:** HealthScorer — THS(Topological Health Score) 다차원 건강도 + Hot/Warm/Cold/Archive 상태 전이
- **Phase 3:** AnomalyCertifier — TAC(Topological Anomaly Certificate) Type A/B/C 분류 + Persistence Score
- **Phase 4:** MetaRuleEngine — Godel Agent 패턴 메타규칙 자기 수정
- DB 추가 테이블: `rule_health`, `exceptions`, `meta_rules`

### 2.5 TCL/TGL 진화 로드맵 (3 Evolution)
- **Evolution 1: RuleGraph** — 규칙 간 위상적 연결 (keyword_overlap + co_activation 엣지), 캐스케이드 활성화
  - DB 테이블: `rule_edges`, `co_activations`
- **Evolution 2: PredictiveTGL** — TGL 간 시간적 선행 패턴 학습, 사전 경고
  - DB 테이블: `tgl_sequences`
- **Evolution 3: AdaptiveTrigger** — 미매칭 학습 + keyword_trigger 자동 확장
  - DB 테이블: `trigger_misses`
  - 검증: "여기까지" 4회 미스 → 자동 확장 → 이후 매칭 성공
- **EnhancedPreflight** — 3가지 진화를 통합한 강화 preflight
- preflight_hook.py에 통합 완료, 7/7 테스트 통과

### 2.6 컨텍스트 압축 (Summary 모드)
- `summary` 필드 추가 — correction_rule 전문을 ~30자 요약으로 압축
- preflight_hook.py에 compact 모드 적용
- **결과: 2,868B → 742B (74% 절감)**, 정보 손실 없음
- 모든 기존 규칙 11건에 대해 수동 최적화된 summary 적용 완료

### 2.7 QMD 연동
- `wesanggoon-sessions` QMD 컬렉션 등록 (E:/AgentInterface/qmd_sessions)
- 컨텍스트 설명 추가 완료
- `sync_rules_to_qmd()` — FTS5 규칙을 QMD 인덱싱 가능 마크다운으로 동기화
- 11건 규칙 동기화 완료

## 3. 기술적 결정 및 근거

| 결정 | 근거 |
|------|------|
| BM25 + 벡터 하이브리드 검색 | 한국어 교착어 특성상 BM25 단독은 형태소 변이에 취약, 벡터는 오매칭 위험 → 둘의 장점 결합 |
| Dynamic RRF 가중치 | 구체적 쿼리(에러코드)는 BM25↑, 추상적 쿼리(전략)는 벡터↑ — Mala et al. 2025 참조 |
| THS 다차원 스코어링 | 단순 시간 기반(9개월)은 드물지만 중요한 규칙을 소멸시킴 → topological_centrality로 보완 |
| UserPromptSubmit Hook | 소프트 트리거(CLAUDE.md 규약)는 잊을 수 있음 → 시스템 강제(하드 트리거) 선택 |
| Summary 압축 vs 초성 압축 | 한국어 초성은 UTF-8에서 바이트 절감 0%, 복호화 정확도 문맥 의존적 → summary가 74% 절감 + 100% 정확 |
| GCE 프레임워크 | 기존 LLM→3DGS 방향만 연구됨, 역방향(3DGS→LLM)은 미개척 — 논문 가치 높음 |
| Qwen2.5-7B 선택 | 4090 24GB에서 FP16 적재 가능(~16GB) + 한국어 성능 우수 + 여유 VRAM으로 학습 가능 |
