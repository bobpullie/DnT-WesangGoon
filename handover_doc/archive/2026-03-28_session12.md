### 2. 과녁 배치 UX v2.0 기획 (모바일 퍼스트) — 정식 채택
- **파일:** `docs/planning/FEATURE_target_placement_ux_v2.md`
- **핵심 설계 결정: Origin-Based Touch Routing**
  - 터치 시작 영역으로 이벤트 100% 구분
  - 슬롯 바 시작 = 아이템 드래그/배치
  - 차트 시작 = 패닝/줌
  - 과녁 위 시작 = 재배치
- **기획안 1: 아이템 슬롯 시스템**
  - 화면 최하단 48px 가로형 슬롯 바, 6슬롯
  - M1: 슬롯1(과녁)만 활성, 나머지 잠금
  - PREDICTING에서만 표시
  - 우측 사이드 패널 제거 → 차트 전폭 확보
  - SUBMIT 버튼 슬롯 바 우측 배치
- **기획안 2: 과녁 배치 UX**
  - 슬롯1 터치+150ms 홀드 → 리프트 → 차트로 드래그 → 드롭 배치
  - 손가락 오프셋 40px (Fat Finger 해결)
  - 재배치: 과녁 직접 터치+드래그 또는 슬롯 재드래그
  - 미배치 30초 만료 → 자동 현재가 배치
- **궤적 통합 방향:**
  - 짧은 모드(3분/5분): 슬롯→드래그 과녁 배치
  - 긴 모드(15분+): 궤적 펜(슬롯3)으로 경로 드래그, 끝점=과녁 자동 배치
  - 아이템 슬롯이 입력 방식을 결정하는 게임적 설계
- **벤치마크:** Clash Royale (카드 슬롯→전장 드래그 배치)
