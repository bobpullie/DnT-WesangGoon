## 2. 완료된 작업 (상세)

### 2.1 실험 환경 구축
- **PyTorch CUDA 설치:** CPU 전용 torch 2.5.1 제거 → torch 2.10.0+cu128 설치
  - CUDA 12.8, RTX 4090 정상 인식, VRAM 14.2GB 사용
- **ML 라이브러리:** transformers 4.47.1, accelerate 1.13.0, peft 0.18.1, bitsandbytes 0.49.2
- **Qwen2.5-7B-Instruct 다운로드:** `E:/models/Qwen2.5-7B-Instruct/` (15GB, 4 shards)
  - Windows symlink 권한 문제로 `--local-dir` 옵션 사용 (HF 기본 캐시 우회)
  - FP16 로드 확인, 한국어 추론 정상 동작

### 2.2 Phase 0: 베이스라인 (TEMS 규칙)
- **파일:** `experiments/phase0_baseline.py`, `src/metrics.py`
- **데이터:** `data/tems_rules.json` (11개 규칙, 732토큰), `data/eval_scenarios.json`
- **태스크:** T1(규칙준수 8건), T2(QA 6건), T3(요약 3건)
- **메트릭:** keyword_match_accuracy, ROUGE-L, TTFT, VRAM
- **결과:** T1=95.0%, T2=60.0%, T3=16.9% (ROUGE-L F1)
- **이슈:** 초기 keyword 매칭이 너무 엄격 → expected_actions를 개별 키워드로 세분화하여 해결

### 2.3 Phase 1: K-means Centroid (TEMS 규칙)
- **파일:** `experiments/phase1_kmeans.py`
- **구현:** `GaussianContext.from_embeddings()` — K-means++ 초기화 + SVD 공분산 피팅
- **이슈 1:** `cdist_cuda` FP16 미지원 → K-means를 FP32로 수행
- **이슈 2:** GaussianContext 생성 시 device 불일치 → `.to(device)` 추가
- **이슈 3:** Splatter의 feature_projector가 랜덤 초기화 → "!!!" 출력 → Phase 1에서는 centroid를 직접 inputs_embeds로 사용하도록 변경
- **결과 (10x 압축):** T1=17.9%, T2=20.6%, T3=13.8%

### 2.4 Phase 2: Differentiable Optimization (TEMS 규칙)
- **파일:** `experiments/phase2_diffable.py`
- **1차 시도:** GaussianSplatter 사용 → NaN loss (relevance 지수함수 오버플로우 + 랜덤 프로젝션)
- **2차 시도:** LearnableContextVectors (K-means centroid를 직접 학습) → Soft Prompt Tuning 방식
- **이슈:** optimizer.zero_grad()가 쿼리 내부 루프에 있어 마지막 쿼리만 학습 → 루프 밖으로 이동
- **최종 설정:** 500 steps, lr=1e-3, 10 training queries, cosine annealing, gradient checkpointing
- **Loss 수렴:** 318 → 32 (90% 감소)
- **결과 (10x 압축):** T1=22.5%, T2=24.7%, T3=16.7%

### 2.5 교과서 데이터 적용
- **데이터셋:** `maywell/korean_textbooks` (HuggingFace) → `ko_wikidata` config
- **선별:** 3개 교과서 텍스트 (스파르타쿠스, 함성대군, 청년 지역사회 참여) = 3,370토큰
- **QA 자동 생성:** Qwen이 교과서를 읽고 10개 QA 쌍 생성
- **파일:** `data/textbook_corpus.json`, `data/textbook_eval.json`, `experiments/textbook_experiment.py`
- **통합 실험 결과 (10x 압축, 3370→337 벡터):**

| Phase | 방법 | QA 정확도 | TTFT |
|-------|------|----------|------|
| Phase 0 | 원본 텍스트 | **86.7%** | 2.17s |
| Phase 1 | K-means centroid | **46.7%** | 1.96s |
| Phase 2 | 학습된 벡터 (100스텝) | **50.0%** | 2.66s |

- **핵심 발견:** 교과서 데이터가 TEMS 규칙보다 훨씬 효과적 (Phase 1: 17.9% → 46.7%)

## 4. 종일군의 핵심 인사이트 (연구 방향에 영향)

### 4.1 "치과 이빨 본뜨기" 비유
- LLM의 임베딩 공간에 맞춰 가우시안을 피팅해야 한다는 직관
- → Phase 2의 KL divergence 역전파 접근법의 근거

### 4.2 "중간 매핑 체계(Encoder-Decoder) 필요성"
- 3DGS의 3D와 LLM의 3584D 사이에 명시적 변환 방정식이 필요
- 현재 Phase 2는 gradient descent로 "알아서 찾게" 하는 방식 (비구조적)
- → Phase 2.5에서 Encoder(토큰→가우시안) + Decoder(가우시안→LLM 입력) 구현 필요

### 4.3 "가우시안으로 LLM 자체를 모사할 수 있는가?"
- 이론적 가능 (Gaussian Mixture는 Universal Approximator)
- NeRF→3DGS 전환의 역사가 선례 (신경망→가우시안 대체)
- 실용적으로는 수십억 가우시안 필요 → 현재는 "입력 압축"에 집중
- → 논문 Future Work 섹션에 기록할 비전

### 4.4 "Soft Prompt = 샘플링 위치를 잘 잡은 것"
- Soft Prompt Tuning의 본질을 한 문장으로 요약
