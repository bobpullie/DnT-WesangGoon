## 1. 세션 요약

Phase 3 Co-training(LLM LoRA + GCE 공동 학습)을 0.5B와 7B 모델에서 실험. 3번의 반복(v1~v3)을 거치며 gradient checkpointing 버그 발견 및 수정. 최종적으로 co-training이 frozen baseline 대비 이점을 보이지 못함 — 0.5B에서는 모델 용량 부족, 7B에서는 LoRA 수정이 생성 품질 저하를 야기.

## 2. 완료된 작업 (상세)

### 2.1 Phase 3 스크립트 작성
- **파일:** `experiments/phase3_cotraining.py`
- **설계:** 2-Phase 학습 (Phase A: GCE warmup, Phase B: LoRA + GCE co-training)
- **기능:** Phase 0 평가, Frozen baseline, Co-training 비교를 한 번에 실행
- **저장:** LoRA adapter + GCE state 자동 저장

### 2.2 0.5B Co-training 실험 (v1~v3)

#### v1 (첫 시도 — 모드 붕괴)
- LoRA rank=16, 8.8M params (모델의 1.75%)
- λ_reg=0.5, lr_lora=1e-4
- target: q/k/v/o/gate/up/down_proj (7개 모듈 전부)
- **결과:** 10.0% (모드 붕괴 — "user user user..." 반복)
- **원인:** LoRA가 너무 공격적 + 정규화 부족 + gradient checkpointing 버그

#### v2 (보수적 LoRA — frozen과 동일)
- LoRA rank=4, 0.27M params (모델의 0.05%)
- λ_reg=3.0, lr_lora=2e-5
- target: q_proj, v_proj만
- 2-Phase: 300 warmup + 700 co-training
- **결과:** 28.3% (frozen과 동일)
- **원인:** gradient checkpointing `use_reentrant=True`가 LoRA 그래디언트 차단 — LoRA가 실질적으로 학습 안 됨

#### v3 (gradient checkpointing 제거)
- v2와 동일 설정, gradient checkpointing 완전 제거 (0.5B는 VRAM 충분)
- **결과:** Frozen 34.2% (이전 28.3%에서 개선!), Co-trained 25.0% (오히려 하락)
- **발견:** gradient checkpointing 제거로 frozen baseline +5.9%p 개선
- **원인:** 0.5B 모델의 intrinsic dimensionality 부족 — LoRA 0.05% 수정만으로도 기존 능력 간섭

### 2.3 7B Co-training 실험
- **설정:** LoRA rank=4, 1.26M params (모델의 0.017%)
- gradient checkpointing: `use_reentrant=False` 적용 (버그 수정)
- 2-Phase: 300 warmup + 700 co-training
- **학습 추이:**
  - Phase A (warmup): 302→34.30 (정상 수렴)
  - Phase B (co-training): 34.31→33.09 (꾸준히 하락, reg_loss 0.001 안정)
- **결과:**
  - Phase 0 (Original): 86.7%
  - Frozen Baseline: 50.0%
  - Co-trained: 34.2% (-15.8%p vs frozen)
- **문제:** 응답에 중국어 혼입 (Qwen 기저 모델 특성), KL loss↓ 이지만 생성 품질↓
- **소요 시간:** ~319분 (5시간 19분)

### 2.4 수학/개념 설명 (종일군 교육)
- LoRA가 "직교 방향"에서 새 지식을 추가하는 원리
- 직교 vs 외적 (3D에서만 유일, nD에서는 SVD로 일반화)
- 고유값(eigenvalue)으로 "빈 방향" 찾기
- K-means의 역할: "씨앗 뿌리기" (계층화가 아닌 거리 기반 그룹핑)
- 3DGS→GCE 파이프라인 전체 흐름: K-means → SVD → 학습

## 3. 핵심 기술적 발견

### 3.1 gradient checkpointing `use_reentrant=True` 버그 (가장 중요)
- **문제:** `model.gradient_checkpointing_enable()` 기본값이 `use_reentrant=True`
- **증상:** "None of the inputs have requires_grad=True. Gradients will be None" 경고
- **영향:** LoRA 파라미터에 그래디언트 미전달 → LoRA가 실질적으로 학습 안 됨
- **해결:** `model.gradient_checkpointing_enable(gradient_checkpointing_kwargs={"use_reentrant": False})`
- **적용 범위:** PEFT/LoRA + gradient checkpointing 조합 시 반드시 필요
- **부가 발견:** 0.5B에서는 gradient checkpointing 자체가 GCE 학습도 방해 (제거 시 +5.9%p)

### 3.2 0.5B 모델 용량 한계
- 0.5B는 모든 파라미터가 "하중 담당" (over-parameterization 없음)
- LoRA 0.05% 수정만으로도 기존 능력 간섭
- Intrinsic dimensionality가 낮아서 직교 방향이 부족
- **결론:** co-training에는 최소 7B+ 모델 필요

### 3.3 KL Loss ≠ 생성 품질
- co-training의 KL loss가 frozen보다 더 낮아졌지만 (33.09 vs 32.84 근접)
- 실제 생성 평가에서는 50% → 34.2%로 하락
- logit 분포 매칭과 실제 텍스트 생성 품질은 다른 목표
- 중국어 혼입 등 generation 특유의 문제가 KL로 포착 안 됨

### 3.4 LoRA rank/target 선택의 영향
| 설정 | 0.5B 결과 | 비고 |
|------|-----------|------|
| rank=16, 7 modules | 10% (붕괴) | 너무 공격적 |
| rank=4, 2 modules | 25~28% | 안정적이지만 효과 없음 |
| 7B rank=4, 2 modules | 34.2% | 중국어 혼입 |

## 7. 종일군 인사이트 메모

- LoRA를 "인터페이스"로 이해 → 디퓨전 모델의 화풍 LoRA와 같은 원리
- "마른 오징어 쥐어짜기" 비유 — 0.5B의 over-parameterization 부족
- 직교 ↔ 외적 관계 질문 → SVD로 일반화되는 원리 이해
- K-means centroid가 "상위개념"이 아닌 "수학적 중간점"임을 이해
- "씨앗 뿌리기 → 자라기 → 꽃피기" 비유로 GCE 파이프라인 전체 흐름 파악

## 8. 핵심 교훈 메모

- **gradient checkpointing + LoRA = 반드시 `use_reentrant=False`:** 기본값이 LoRA 그래디언트를 차단
- **0.5B는 co-training 부적합:** intrinsic dimensionality 부족으로 LoRA가 기존 능력과 간섭
- **KL loss ≠ 생성 품질:** logit 분포 매칭이 좋아져도 텍스트 품질은 나빠질 수 있음
- **Qwen 모델의 중국어 편향:** LoRA 수정 시 중국어 모드로 되돌아가는 경향
- **co-training은 더 많은 학습 데이터가 필요:** 10개 QA로는 LoRA가 의미있게 학습하기 어려움
