## 2. 완료된 작업 (상세)

### 2.1 Phase 2.5 v1: 거대 Encoder (실패 → 0%)
- **파일:** `src/encoder.py`, `src/decoder.py`, `src/pipeline.py`, `experiments/phase25_encoder_decoder.py`
- **설계:** Perceiver-style Cross-Attention (M queries attend to N tokens) + FFN + 파라미터 헤드(μ, Σ, α, f)
- **파라미터:** 248M → 161M (bottleneck 추가 후)
- **결과:** 0% 정확도 — 출력 "Assistant Stewart" 등 gibberish
- **원인:** 161M params에 6쿼리 × 300스텝 = 심각한 과소적합. Cross-Attention + LayerNorm이 K-means centroid의 LLM 임베딩 분포를 파괴.

### 2.2 Phase 2.5 v2: Warm-start + LayerNorm 제거 (실패 → 0%)
- K-means centroid로 latent_queries 초기화
- Decoder의 LayerNorm 제거
- 임베딩 분포 매칭 loss (dist_loss) 추가
- **결과:** 0% — α가 0으로 붕괴, feature 보정 비활성화
- **원인:** 여전히 Cross-Attention이 분포 파괴

### 2.3 Phase 2.5 v3: Gated Residual (실패 → 3.3%)
- gate_attn = sigmoid(-5) ≈ 0.007로 시작, cross-attention 신호를 점진적으로 개방
- LayerNorm 완전 제거
- **결과:** 3.3% — gate가 0.007에서 0.0068로 거의 변하지 않음
- **원인:** 101M params에 학습 신호 부족, gate가 열리지 않음

### 2.4 Phase 2.5 v4: 경량 가우시안 구조 (성공! → 56.7%)
- **핵심 전환:** 큰 Encoder 포기, Phase 2 위에 가우시안 구조를 얹는 접근
- **구조:** `v = μ + α · proj(f)` — means(직접 학습) + opacity × feature_projection
- **파라미터:** 7M (vs 이전 161M)
- **초기화:** K-means centroid → means, feature_proj = zeros → 초기에 Phase 2와 동일
- **NaN 버그 발견 및 수정:**
  - `GaussianContext.from_embeddings()`에서 SVD 계산 시 `S.sum()==0`인 클러스터 → features에 NaN
  - IEEE 754: `0 * NaN = NaN` → feature_proj 가중치가 0이어도 NaN 전파
  - 해결: `torch.nan_to_num(feat_data, nan=0.0)`

### 2.5 Step Sweep (12쿼리, 10x 압축)
- **파일:** `experiments/phase25_sweep.py`
- 학습 쿼리를 6개 → 12개(QA 10개 + 일반 2개)로 확대
- 결과:

| Steps | QA 정확도 | KL Loss | 시간 |
|-------|----------|---------|------|
| 300 | 40.8% | 34.39 | 7분 |
| 500 | 43.3% | 33.56 | 12분 |
| **1000** | **56.7%** | **32.84** | **24분** |
| 2000 | 50.0% | 32.42 | 47분 |

- **Sweet spot: 1000스텝** — 2000에서 과적합 발생 (loss↓ but accuracy↓)

### 2.6 압축률 실험 (5x)
- 5x, 1000스텝 → 53.3% (10x 56.7%보다 낮음)
- **원인:** 674벡터(13M params)에 같은 1000스텝 = underfitting
- 5x는 더 많은 스텝이 필요 (미실행)

### 2.7 α 초기값 실험
- α=0.01(K-means 비율) → 46.7% (feature 보정 비활성)
- α=0.5(균일) → 50~56.7% (feature 보정 활성화)

## 3. 기술적 결정 및 근거

| 결정 | 근거 |
|------|------|
| 큰 Encoder → 경량 구조 전환 | 161M params + 6쿼리 × 300스텝 = 파라미터당 학습량 0.01x. Phase 2(1.2M)는 1000x. 학습 신호에 맞는 모델 크기 필수 |
| LayerNorm 제거 | LLM 임베딩의 실제 분포(스케일/시프트)를 보존해야 LLM이 이해. LayerNorm이 이를 정규화하여 파괴 |
| feature_proj = zeros 초기화 | 초기 상태에서 v = μ = Phase 2와 동일 → 안정적 출발점. 점진적으로 feature 보정이 기여 |
| α = 0.5 균일 초기화 | K-means 비율(0.01)이면 feature 보정 불가. 0.5로 시작하여 학습이 조절하게 함 |
| NaN → nan_to_num | SVD에서 빈 클러스터의 features가 NaN. 0 * NaN = NaN (IEEE 754). 치명적 버그 |
| 12쿼리로 확대 | 6쿼리보다 풍부한 학습 신호 → 56.7% (6쿼리 53.3% 대비 +3.4%p) |
| 1000스텝 = sweet spot | 2000에서 과적합 (loss↓ accuracy↓). 적절한 정밀도 + 유연성 균형 |

## 4. 종일군의 핵심 인사이트 (연구 방향에 영향)

### 4.1 "Soft Prompt의 본질" 감상
- 연속적 의미 공간에서 이산적 토큰은 "핀" → Soft Prompt는 핀 사이를 걸어다니며 최적 위치 탐색
- 언어란 머릿속의 연속 의미를 이산 채널로 압축한 것 → GCE는 이 역과정

### 4.2 "Co-training" 아이디어 (Phase 3 확정)
- 현재(Post-hoc): frozen LLM + GCE = "통역사를 통한 소통"
- Co-training: LLM + GCE 함께 학습 = "이중언어로 교육"
- Qwen2.5-0.5B를 GCE와 공동 미세조정하면 압축 보존율이 근본적으로 향상 예상
- 4090 하나로 4~8시간이면 가능

### 4.3 실시간 오버헤드 질문
- GCE 인코딩+디코딩: ~2-3ms (무시 가능)
- 진짜 절약: LLM attention 토큰² → (토큰/10)² = 100배
- 3370토큰은 너무 작아서 현재 실험에서 TTFT 차이 없음
- 50K+ 토큰부터 GCE의 가치 극대화

## 8. 핵심 교훈 메모

- **학습 신호에 맞는 모델 크기:** 161M params + 1800 gradient updates = 0.01x per param → 실패. 7M + 12000 updates = 1.7x → 성공
- **분포 보존이 최우선:** LayerNorm, Cross-Attention 등이 LLM 임베딩 분포를 파괴하면 0%. K-means centroid에서 시작하여 점진적 보정이 핵심
- **IEEE 754: 0 × NaN = NaN:** SVD에서 빈 클러스터의 NaN이 feature_proj(zeros) × features(NaN) = NaN으로 전파. nan_to_num 필수
- **과적합의 양상:** KL loss는 계속 내려가지만 정확도는 1000스텝에서 피크. 학습 분포 매칭과 생성 품질은 다른 목표
- **Co-training이 근본적 해법:** Post-hoc 압축은 한계(~65% 보존). LLM이 가우시안을 "모국어"로 배우면 보존율 극대화 가능
