## 이번 세션의 고질적 문제 케이스 (다음 세션 논의용)

### 케이스 A: 코드 정합성 ≠ 비주얼 정합성
- 기획군/빌드군 코드 리뷰에서 PASS 판정 → 실제 화면에서는 미적용
- 원인: 에이전트가 화면을 볼 수 없어 "코드가 맞으면 작동할 것"으로 추정
- 해결 시도: AVIS 구축 (스크린샷 캡처 + Claude Read 분석)
- **미해결:** Gemini API 빌링 문제로 자동화 검증 루프 미가동

### 케이스 B: 파라미터 변경이 실제 적용되지 않는 문제
- shared 패키지 빌드 누락 (dist에 반영 안 됨)
- dev 서버 HMR이 신규 모듈 import를 인식하지 못함
- **교훈:** 파라미터 변경 후 반드시 `cd shared && npm run build` + dev 서버 재시작

### 케이스 C: 차트 좌표계 혼동
- timeScale(가로 줌) vs priceScale(세로 줌) 구분 실패 반복
- scaleMargins 방향 오류 (여백 확대 = 캔들 축소, 역방향)
- logicalRange.to ≠ 실제 데이터 마지막 인덱스 (미래 여백 포함)
- **TGL#45 등록:** 차트 세로 크기는 priceScale 조절

### 케이스 D: alignToTimeframe과 useTargetZoom 충돌
- CandlestickChart의 `alignToTimeframe`(PREDICTING용)과 `useTargetZoom`(LIVE용)이 독립적으로 차트 범위를 설정
- PAST_MULTIPLIER=30.0이 너무 커서 PREDICTING 정렬이 과도하게 줌아웃

### 케이스 E: 이모티콘 비주얼 명세 오류
- 기획 파라미터 자체가 잘못됨: `rgba(0,0,0,0.4)` = 검은 배경에 검은 라인
- 구현 버그가 아닌 명세 오류 → 기획군이 비주얼 파라미터를 정할 때 배경 색상 고려 필요

### 케이스 F: 화면 정렬 후 세로 패닝 잠김
- lightweight-charts에서 프로그래밍 방식으로 줌/정렬 변경 시 autoScale이 활성화
- 유저가 가격눈금을 터치해야 해제됨
- **TGL#46 등록**
