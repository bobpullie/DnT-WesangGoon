# Session 31 — TEMS 표준화 원복 + Phase 0/0.5/0.6/1/2 구현 (TEMS Systematizing)
> 2026-04-19

## 목표
종일군이 "최근 표준화한 TEMS에 오류가 많다"고 원복 요청 → 위상군 자체 TEMS를 self-contained로 복원하고, 그 과정에서 드러난 4가지 구조적 사각지대를 4개 Phase에 걸쳐 해소. TEMS를 "데이터베이스"에서 "규칙 시스템"(Systematizing)으로 격상.

## 수행 내용

### 1. 위상군 자체 TEMS 표준화 원복 (S29 → 원본)
- **원인:** S29 표준화로 `from tems.fts5_memory` 등 외부 패키지 import로 변경. `tems` 패키지가 site-packages 미배포 → ModuleNotFoundError → preflight/tems_commit/backfill 모두 silent fail
- **복원:** `_backup_tier1/`에서 `fts5_memory.py`, `tems_engine.py`, `tems_commit.py`, `preflight_hook.py` 복원. `__init__.py`, `backfill_triggers.py`를 로컬 import로 전환
- **보존:** `_post_S29_standardization/`에 표준화 버전 4파일 유지 (외부 패키지 정식 배포 시 재참조)
- **TGL #92 등록:** 외부 패키지 마이그레이션 시 import 검증 + self-contained 백업 보존 필수

### 2. TEMS 구조적 결함 진단 (4개 사각지대)
- A. 트리거가 사용자 프롬프트 키워드에만 결합 — agent의 도구 결과/상태 무관찰
- B. Truncation으로 가드 행동 절반 손실
- C. 강제 메커니즘 부재 (honor-system)
- D. 세션 종료 회고 강제 hook 없음

### 3. TEMS Phase 0 (자기관찰 + 충실도)
- **T1.1** `preflight_hook.py` `except:pass` 제거 → 구조화 로깅 + `<preflight-degraded>` 출력 + `memory/tems_diagnostics.jsonl` 영속화
- **T1.2** `memory/tool_failure_hook.py` 신규 — Bash PostToolUse에서 11개 실패 시그니처 감지 → `<tool-failure-detected>` + `memory/tool_failures.jsonl`
- **T2.2** TGL 풀텍스트 주입 (truncation 제거). TCL은 summary 유지
- **TCL #94 등록**

### 4. TEMS Phase 0.5 (조용한 TEMS — 종일군 지시)
- 매 prompt 무차별 주입 금지 → 키워드 강매칭 시에만 발동
- `SCORE_THRESHOLD=0.55` 도입, `stopwords` 확장 (테스트/확인/진행 등)
- CURRENT_STATE.md 상단에 TEMS 호출 매뉴얼 1회 주입 (SessionStart)
- **TCL #95 등록**

### 5. TEMS Phase 0.6 (Dense semantic 부활 — 종일군 정정)
- 위상군이 이미 구축한 CUDA QMD dense + HybridRetriever를 잊고 reinvent했음을 종일군이 지적
- `SEMANTIC_FALLBACK_ENABLED=True` 복원
- `qmd embed` 신규 4규칙 임베딩 → `tems-wesanggoon` 컬렉션 활용
- 검증: "패키지 디펜던시 사라지면" → BM25 0개 매칭이지만 dense로 TGL #92 정확 발동
- **TGL #96 등록:** "위상군 자신이 이미 구축한 시스템 잊고 reinvent 금지"

### 6. TEMS Phase 1 (자가진화)
- **T1.3** `memory/pattern_detector.py` 신규 — jsonl 스캔, 정규화, 그룹화, 반복 임계 이상 후보 추출
- **T4.1** `memory/retrospective_hook.py` 신규 — Stop event hook, RATE_LIMIT_SEC=600 throttle
- 모드 토글: `is_auto_register_enabled()` — TCL "자동등록 활성화" 존재 시 자동등록
- 중복 방지: `is_already_registered_pattern()` — auto-detected 태그 + signature
- **TCL #97 등록 (종일군 활성화):** TEMS 자동등록 활성화
- **TGL #98 자동등록 (Phase 1 첫 자율 산출물):** python_traceback ×7회 (false-positive였으나 메커니즘 정상 입증)
- **TCL #99 등록**

### 7. TEMS Phase 2 (분류·템플릿·게이트·카운팅 — Systematizing)
- **2A** `rule_health` 스키마 확장: `fire_count`, `last_fired`, `compliance_count`, `violation_count`, `classification`, `abstraction_level`, `needs_review`, `created_at`. preflight가 cap 후 실제 주입 ID만 fire_count++
- **2B** TCL/TGL 재정의 (CLAUDE.md, tems-protocol.md):
  - TCL = 좁은 규칙 (BM25만, 키워드 다양성 게이트)
  - TGL = 넓은 위상 패턴 (dense + 7-카테고리 + 템플릿)
- **2C/D/E** `tems_commit.py` 통합 재작성:
  - 7-카테고리 자동 분류 (TGL-T/S/D/P/W/C/M)
  - 게이트 A(schema) B(abstraction) **거부형** / C(duplication) D(replay) E(verifiability) **경고형**
  - 12개 새 슬롯: `--classification`, `--abstraction`, `--topological-case`, `--forbidden`, `--required`, `--semantic-intent`, `--failure-signature`, `--tool-pattern`, `--success-signal`, `--compliance-check`, `--needs-review`, `--dry-run`
- **TGL #98 archive → TGL #100 보강 재등록:** TGL-M/L2 첫 시범 — "관찰 시스템 self-trigger 루프 방지"
- **TCL #101 등록**

## 미완료 / 다음 세션 (S32) Task

### P0 — TEMS Phase 3 (강제 메커니즘)
- **TGL-T용 PreToolUse hook**: 각 TGL-T 규칙의 `tool_pattern` 정규식 매칭으로 위험 도구 호출 사전 차단/경고
- **Compliance/Violation 자동 측정**: TGL 매칭 후 위상군의 다음 N개 도구 호출 추적 → `forbidden_action` 위반 시 `violation_count++`, 회피 시 `compliance_count++`. rule_health에 `efficacy_score` 자동 갱신
- **30/90일 자동 cold/archive 전환 cron**: `fire_count=0 in last 30d → cold`, `90d → archive` 후보

### P1 — 기존 규칙 마이그레이션 + 자가관찰 보강
- **기존 66개 규칙 자동 분류**: rule_health에 classification/abstraction 채우기 (텍스트 분석 기반)
- **TGL #98(archive) 외 자동등록 누적분 재검토**: 같은 시범 패턴으로 보강 필요한 규칙 식별
- **false-positive 보강**: `tool_failure_hook.py`의 `IGNORE_PATTERNS`에 다음 추가
  - `cat .*\.jsonl`, `tail .*\.jsonl`, `head .*\.jsonl` (자기 로그 읽기)
  - `python memory/.*hook\.py` (자기 hook 실행)
  - `python memory/(pattern_detector|retrospective_hook|tems_commit)` (자기 도구 실행)
  - `echo .*Traceback`, `echo .*ModuleNotFoundError` (테스트 페이로드)
- **Stop hook이 needs_review 표시 후보 알림**: L1 자동 등록 사례를 위상군이 L2로 보강할 수 있도록 retrospective_hook이 needs_review=1 규칙 N개를 함께 표시
