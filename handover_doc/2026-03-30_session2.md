## 1. 이번 세션 논의/결정 요약

### 1-1. 캐릭터 커스터마이징 → 선택 방식 피벗 (종일군 결정)
- 현실적 제약으로 5부위 파츠 조합(1,536종) → **기본 8캐릭터 프리셋 선택**

- 성능 최적화: 선택창은 정지샷만 로드 → 선택 후 풀 스프라이트 lazy-load

### 1-2. 캐릭터카드 세로형 전환 (종일군 결정)
- 캐릭터 최소 128x128px 필요 → 128x160px (4:5 비율) 세로형 카드
- SidePanel 하단 UserCard: 160px → 252px
- 상단 콘텐츠 잔여: 423px (최소 400px 충족)

### 1-3. 차트-캐릭터 전투 모션 연계 시스템 (종일군 아이디어 → 기획)
- LIVE 상태에서 캐릭터가 차트 가격 변동에 반응하여 전투 모션 표시
- Phase C: emotionEngine(Phase B)과 독립적인 battleMotionEngine
- v1.0 → v2.0 진화: 모션 전환 조건 전면 재설계

### 1-4. 모션 전환 로직 v2.0 (종일군 피드백 반영)
- attack → light_attack + heavy_attack 2단계 분리
- walk: 밴드 안 경계 근처 / dash: 밴드 안 깊이 진입
- guard: hit 누적이 아닌 "밴드 접근 중" 방향 판정으로 변경
- 히스테리시스 ±0.05 (flickering 방지)
- hit 빈도 제어: 쿨다운 1.5초 + 10초 내 3회 상한

### 1-5. 아트 파이프라인 R&D (아트군 HO-011)
- PixelLab → ComfyUI v8c TO 스타일 변환 파이프라인 확정
- 커스텀 모션 필수 (템플릿은 범용 차렷 자세, 장비 특화 불가)
- 동적 프레임 수 허용 (REQ-001 승인, JSON 메타데이터 기반)
- 리아(여전사) 1차 테스트 캐릭터로 전투 스프라이트 v3 제작 완료
