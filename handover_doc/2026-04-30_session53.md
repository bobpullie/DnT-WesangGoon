## 3. Viewer 패치 상세

### 3.1 D1 fix — `__mp_main__` 가드 제거

```python
# Before
if __name__ in ("__main__", "__mp_main__"):
    sys.exit(main())

# After
if __name__ == "__main__":
    # __mp_main__ 제거 — NiceGUI native_mode 가 mp.Process 로 _open_window 자식 spawn 시
    # 자식이 이 스크립트를 spawn import 하면 (실제로는 nicegui.native.native_mode 만 import 하지만,
    # 가드 누수 방지) main() 재진입 → ui.run(native=True) 중첩 호출 → 윈도우 즉시 종료. S52 진단.
    sys.exit(main())
```

검증 path:
- 1단계: foreground `timeout 8 python viewer/rule_viewer.py` → Chromium teardown error (`window_impl.cc(121) Failed to unregister class Chrome_WidgetWin_0. Error = 1411`) — 윈도우는 떴음
- 2단계: subprocess `stdout=DEVNULL` → `[NiceGUI ready]` 후 rc=0 (2 초). stdin 영향이라 가설
- 3단계: stdin 까지 inherit → 같은 패턴 (rc=0 2 초)
- 4단계: smoke 테스트 (`@ui.page('/'); ui.run(native=True)` top-level, no main guard) → 12 초 생존 (성공)
- 5단계: 동일 viewer 코드를 main guard 만 `__main__` 으로 → 8 초 생존 (성공). **확정**

`mp.Process(target=_open_window, args=args, daemon=True)` 의 daemon 자식이 fresh import 하면서 viewer 의 `__mp_main__` 가드도 자식에서 매칭 → main() → ui.run native 재진입 → 즉시 종료.

### 3.2 D2 fix — pythonw.exe + CREATE_NO_WINDOW

[.claude/commands/tems-view.md](.claude/commands/tems-view.md) 의 launcher 코드:

```python
exe = sys.executable
if os.name == 'nt':
    pw = Path(sys.executable).with_name('pythonw.exe')
    if pw.exists(): exe = str(pw)

```

### 3.3 D3 fix — DOM chain dump 진단 + flex chain 강제

진단 단계 (TCL #129 적용):
1. NiceGUI/Quasar 의 served HTML 만으로는 SPA 라 DOM 안 보임 (49KB shell + JSON config)
2. 임시 JS dump 코드 추가 (`async def _dump_dom_chain()` + `ui.run_javascript`) → q-table__container 부터 body 까지 parent chain walking + 각 layer 의 `{display, flexDirection, flex, overflow, height, minHeight, getBoundingClientRect.h}` 채취 → `logs/dom_chain.json` 적재
3. dump 결과 분석:

| depth | element | h (px) | display | 문제 |
|-------|---------|--------|---------|------|
| 0 | `q-table__container.q-table__card` | 2498 | flex column ✅ | 모든 행 자연 높이 (101 rule × 페이지 50) |
| 1 | `nicegui-content` | 2570 | flex column | 무한 grow |
| 2 | `MAIN.q-page` | 2570 | flex column | 무한 grow |
| 3 | `q-page-container` | 2706 | **block** ❌ | **flex chain 단절** |
| 4 | `q-layout.nicegui-layout` | 2706 | **block** ❌ | **flex chain 단절** |
| 5 | `<div>` (body 직속, no class — Vue mount wrapper) | 2706 | block | |
| 6 | `BODY` | 763 | block, overflow:hidden | viewport (763px) |

확정: q-layout / q-page-container 가 Quasar default 로 `display: block`. body 의 overflow:hidden 은 외곽 스크롤은 차단했지만 내용을 자르기만 했고, q-table 은 무한 grow 한 채로 pagination 이 viewport 밖으로 밀림.

Fix CSS ([viewer/rule_viewer.py:443-462](viewer/rule_viewer.py#L443-L462)):

```css
html, body { height: 100%; margin: 0; overflow: hidden; }
body > div { height: 100vh; display: flex; flex-direction: column; min-height: 0; }
.nicegui-layout, .q-layout { height: 100vh !important; display: flex !important; flex-direction: column !important; min-height: 0 !important; }
.q-page-container { flex: 1 1 auto !important; min-height: 0 !important; display: flex !important; flex-direction: column !important; }
.q-page-container > .q-page { flex: 1 1 auto !important; min-height: 0 !important; display: flex !important; flex-direction: column !important; }
.q-page > .nicegui-content { flex: 1 1 auto !important; min-height: 0 !important; padding: 0 !important; }
.q-table__container { min-height: 0; }
.q-table__middle { flex: 1 1 auto; min-height: 0; overflow: auto; }
.q-table__top, .q-table__bottom { flex: 0 0 auto; }
```

검증 (두 번째 dump):

| element | h (px) before | h (px) after |
|---------|--------------|-------------|
| q-table__container | 2498 | **555** |
| nicegui-content | 2570 | **627** |
| q-page | 2570 | **627** |
| q-page-container | 2706 | **763** |
| q-layout | 2706 | **763** |
| body > div | 2706 | **763** |
| body | 763 | 763 (viewport) |

모든 chain 이 viewport 763px 안에 fit. q-table__container 555px = (763 - header 64 - footer 72 - filter 50 - margins).

debug dump 코드는 fix 검증 후 제거 (PID 41608 = 깨끗 버전).
