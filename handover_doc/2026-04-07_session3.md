### 2. 🔴 플랜 결함 발견 및 근본 해결: `.scripts/` → `khub/` rename
**문제:** 플랜 전체가 `.scripts/` (선행 점) 디렉토리를 Python 패키지로 사용. 이는 PEP 517/setuptools 검증에서 두 가지로 거부됨:
1. `[project.scripts] k = ".scripts.cli:cli"` — entry-point는 점으로 시작 불가
2. `import .scripts` 자체가 Python에서 불가 (relative import 토큰)

**증거:** 종일군도 동일한 pip 설치 실패로 고생 중이었음.

**해결:**
- 플랜 파일 전체에서 `.scripts` → `khub` 일괄 치환 (sed, 131건)
- 디스크 디렉토리도 rename
- `pyproject.toml`에 `[tool.setuptools] packages = ["khub"]` 명시 추가 (자동탐색 실패 방지)
- sed가 `[project.scripts]` 헤더를 `[projectkhub]`로 오염시킨 부분 별도 복구
- git: `c7785c3 fix: rename .scripts package to khub for PEP 517 compliance`

**미반영:** 원본 spec 문서 `E:/DnT/DnT_WesangGoon/docs/superpowers/specs/2026-04-07-knowledge-hub-design.md` 는 아직 `.scripts/` 표기 그대로일 가능성 — Phase 2 시작 전 동기화 필요.

### 4. Reviewer 지적 사항 (deferred to backlog)
크리티컬 0건. 다음 항목은 spec 준수상 변경 보류했으나 Phase 2 이후 고려:
- **`generate_node_id` 충돌 위험:** 6 hex chars = 16M space, birthday collision ~50% at 4,800 IDs/type/day. → `token_hex(4)` 권장
- **tags split-on-space 제약:** `tags.split(" ")` — 공백 포함 태그 불가, 이중 공백 시 빈 문자열. 스키마 레벨 제약으로 굳어짐
- **CRLF 처리 부재:** `frontmatter_io.parse_file`이 `startswith("---\n")` 정확히 LF만 검사. Windows 작성 파일이 CRLF면 false-raise
- **`_row_to_node` dict 검증 부재:** `yaml.safe_load`가 list/scalar 반환 시 무방어
- **`Edge.created_at` 기본값 None:** `field(default_factory=datetime.utcnow)` 권장
- **dict 타입 힌트 unparameterized:** `raw_frontmatter: dict` → `dict[str, Any]`
- **`_row_to_edge` 내부 datetime import:** 모듈 top으로 이동 권장
- **`get_neighbors` N+1 쿼리:** 깊이 큰 BFS에서 perf 이슈 가능
