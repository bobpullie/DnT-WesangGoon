## 이번 세션의 교훈 (Lessons)

1. **SDC 위임의 실제 break-even이 생각보다 높다** — 1-2줄 편집은 직접 처리가 우월. Gate의 Q2(Overhead) 판정이 실무상 가장 빈번.
2. **Nested git repo 처리 필수** — 위상군 프로젝트 안에 DVC clone 이 있으면 root `git status`가 오염. `git rm --cached` + `.gitignore` 로 경계 처리.
3. **Auth 전환 관리** — gh 계정 여러 개 있을 때 `gh auth switch` + `setup-git` 필요. 레포 URL만 바꿔서 push하면 credential helper가 엉뚱한 계정으로 시도.
4. **Sonnet subagent가 "기존 페이지 수정 금지" 제약을 과도 해석** — TWK rename 시 Sonnet이 `System_vs_Skill.md`를 건드리지 않았는데, 그 페이지는 이 세션에서 막 생성한 것이라 여전히 기록 업데이트 필요성이 있었음. Brief에서 "이미 작성된 wiki 기존 페이지" 조건을 더 정밀하게 써야.
5. **hybrid trigger 가 full-check보다 실용적** — 매 요청 게이트는 overhead, 대부분 일반 대화는 게이트 불필요. Trigger 기반이 실제 절감과 부담 완화의 sweet spot.
