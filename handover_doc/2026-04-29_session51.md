### 4.3 codex 호출 함정 (위상군 환경 한정)
codex 0.125.0 + Windows + git bash + non-TTY 조합에서 stdin handling 함정. 4종 시도:
1. git bash + `--full-auto` + stdin pipe → 무한 hang (codex 가 stdin EOF 못 받음)
2. PowerShell tool + Tee-Object + stdin pipe → "Reading additional input from stdin..." 후 hang
3. PowerShell + `Start-Process -RedirectStandardInput "NUL"` → NUL device path 해석 실패
4. PowerShell + `Start-Process -FilePath "codex"` (codex.cmd 직접 실행) → "%1은 올바른 Win32 응용 프로그램이 아닙니다" (TGL #69 정확 매칭 — 위상군이 첫 응답에 misclassify 함, 메타-결함)
5. **PowerShell + `cmd /c "codex ... < prompt.txt > log.txt 2> err.txt"` → 정착**

→ S52+ TGL 등록 후보: "Windows + non-TTY 환경에서 codex CLI 호출 시 cmd.exe wrapper + stdin file redirect 강제. PowerShell `Start-Process` 직접 실행 금지 (codex.cmd 인식 실패)".

## 10. 반성

1. **TGL #69 misclassification = 본 세션의 자기 인지 결함.** preflight 가 정확히 cite 했는데 응답 첫 줄에 "무관, skip" 단언. 직후 시도가 정확히 TGL #69 의 케이스로 실패. 위상군이 정정만 하고 TEMS 등록 안 함 — Layer D 가 잡아야 할 정확한 시그널을 본 세션 안에서 만들고 다음 세션 검출 대상으로 넘김.
2. **Spec v1 의 본질적 결함 4건을 audit 가 사전 차단.** Layer C 논리 오류 (chat 텍스트 검사 → Edit/Write payload 검사), self-loop 불완전, whitelist 얇음, transcript schema 누락. audit 없이 codex 직행했다면 구현 결함 누적. 위임 모델의 audit 단계가 메타-결함 차단에 critical 함을 입증.
3. **codex 호출 진단 4종 시도 = 위상군 환경 함정 학습.** Windows + git bash + non-TTY + npm .cmd wrapper 의 조합. 종일군 PS terminal 검증 (즉시 응답) → 위상군 환경 한정 문제 식별 → cmd.exe wrapper 정착. S52 TGL 등록 의무.
4. **위상군이 직접 구현 안 함 = 메타-결함 자기-방어 동기 구조적 차단.** S50 의 "smoke test 통과 직후 자축" 패턴 본 세션 0건. 구현자 ≠ 검증자 → 자축 동기 0. 검증 톤이 자연스럽게 비판적 (codex 자가 18 acceptance 보고 → 위상군 직접 6 검증).
5. **spec v2 본문에 자축어 / 절대화 0건 명시 점검.** "완벽", "ㅋㅋ", "전면 강제", "100%" 등 본 hook 이 잡을 토큰 본 spec 안에 있으면 메타-부조리. 점검 완료.
6. **본 핸드오버 본문 자축 점검.** 본 spec 의 self-cognition gate 가 다음 세션 SessionStart 시 본 핸드오버 작성 turn 을 검사할 것. 자축 토큰 0건 명시 회피. 객관 단언 + 한계 명시 + 미해결 항목 cite.
