## 핵심 논의 및 결정

### 1. Atlas의 본질 (종일군 정의)
- Atlas는 **에이전트의 외부화된 작업 기억(externalized working memory)**
- 사람용 문서가 아닌, 에이전트가 프로젝트를 추상화 계층으로 경량 추적하는 시스템
- 세 가지 역할 동시 수행:
  - **렌즈** — 에이전트가 프로젝트를 이해하는 도구
  - **설계 관문** — 코드 수정 전 반드시 거치는 절차
  - **경험 저장소** — 버그/수정/교훈이 축적되는 히스토리

### 2. `/atlas` 명령 목적
- **C: 초기 셋업 마법사 + 기존 프로젝트 backfill 통합**
- 프로젝트 규모가 커지는 시점에 의도적으로 "활성화"하는 행위
- 에이전트가 임의로 만든 문서가 아닌, atlas 스킬이 만든 문서임을 사람이 확인 가능 (진정성 보증)

### 3. 사람 확인 최소화 결정 (핵심)
- 종일군은 개발자 출신이 아님 → 기술적 세부(모듈 분류 등) 검증 불가
- Confirm Gate를 사람에게 두면 오히려 문서 품질 저하
- **결론: Autonomous + Self-Validating** 설계
  - 사람이 하는 건 `/atlas` 입력 하나
  - 나머지는 결정론적 규칙 + 자동 검증 + 자동 수정
  - 3중 안전장치: (1) 결정론적 규칙 (2) atlas check self-validate (3) rollback 가능

### 4. 접근법 선택
- **Approach C: 단일 스킬 + 명시적 Phase 구조**
- SKILL.md 하나로 Part A (활성화) + Part B (작업 루프) + Part C (히스토리) 통합
- 배포 메커니즘(init-skill)이 단일 SKILL.md 기준으로 설계되어 있어 자연스러운 업그레이드
