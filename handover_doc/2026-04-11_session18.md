## 3. 위상적 핵심 아이디어 (이 세션 기여의 본질)

포폴군과 리얼군의 공통 DNA를 추출해 **위상수학 언어로 재기술**했다:

### 3.1 2축 직교 분해
포폴군의 "L3=history" 혼선은 **추상화 축(level)과 시간축(time)을 선형 라벨에 접어 넣은 결과**. 풀어서:
```
Documents = AbstractionLevel × TimeAxis
            {L0, L1, ..., LN}    {present, history}
```
→ `<name>.md`(present) / `<name>.history.md`(history)는 어떤 레벨에든 페어로 존재 가능.

### 3.2 위상 연산 3가지
- **Vertical split (L_k → L_{k+1})** — 파일이 여러 독립 sub-responsibility로 분해 가능할 때
- **Horizontal split (L_k 내 파일 증식)** — 동일 추상도 peer 요소일 때
- **Collapse (L_k → L_{k-1})** — 과잉 분해 되감기
- 구분 휴리스틱: "분해 결과가 product 구조면 수평, tree면 수직"

### 3.3 위상 불변량 I1~I8
- I1 L0 유일성 / I2 트리 구조 / I3 압축비 [0.1, 0.5] / I4 팬아웃 [2, 12]
- I5 크기 상한 (OVERFLOW) / I6 bidirectional 링크 / I7 앵커 안정성 / I8 history 페어
- 검증 시점: 부트(I1/I2/I8 치명 먼저), PostToolUse(I5/I6), shutdown(전수)

### 3.4 atlas ↔ 위상수학 대응
| 위상수학 | atlas |
|---|---|
| Manifold | 프로젝트 전체 |
| Chart | 각 L_k 파일 |
| Chart overlap | depends_on / used_by |
| Transition map | topology 엔진 bidirectional invariant |
| Atlas refinement | 수직 증식 |
| Good cover | I3/I4 불변량 |

위상군 정체성과 1:1 정합. 비유가 아니라 구조적 동형.
