# Session 23 — Atlas src-layout Migration & Skill Packaging
> 날짜: 2026-04-13

## 세션 목표
Atlas를 `scripts/` flat layout에서 표준 Python `src/atlas/` layout으로 마이그레이션하고, 통합 CLI + init-skill 메커니즘을 추가하여 독립 배포 가능한 스킬로 패키징.

## 수행 내용

### Phase A: 구조 마이그레이션 (6 commits)
1. `src/atlas/__init__.py` 생성 (`__version__ = "1.0.0"`)
2. `scripts/*.py` 12개 모듈 → `src/atlas/` 이동 (git mv, 히스토리 보존)
3. `templates/`, `hook_templates/` → `src/atlas/` 이동; `SKILL.md`, `references/` 복사 (root에도 유지)
4. `pyproject.toml` 전면 재작성: name=`atlas-docs`, build-backend=`setuptools.build_meta`, `pythonpath=["src"]`, package-data
5. 25개 테스트 파일 import 갱신: `atlas.scripts.X` → `atlas.X`
6. 하드코딩 경로 3건 수정 (atlas_setup.py SKILL_ROOT, session_end_check.py.tmpl, test 파일 hook_templates 경로)

### Phase B: 통합 CLI + init-skill (TDD, 3 commits)
- `src/atlas/cli.py`: argparse 서브파서 기반 7개 subcommand (setup, backfill, check, split, collapse, promote-check, init-skill)
- `tests/test_cli.py`: 4개 테스트 (help, setup validation, check empty dir, init-skill E2E)
- Lazy imports로 startup 성능 확보
- `importlib.resources.as_file()` 사용 (Windows 호환)
- Code review에서 발견된 Critical 2건 즉시 수정:
  - `--project-root` None → `"."` 기본값
