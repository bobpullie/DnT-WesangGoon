## 논의 및 결정

### 1. 시작점 — 종일군의 핵심 고통
종일군이 노션에 모든 사고/메모를 기록해왔으나:
- 여러 세션·프로젝트가 한 페이지에 섞여 알아보기 힘듦
- 작업 끝나면 데이터·노하우가 사라지는 느낌
- **값진 경험의 디테일이 핸드오버 요약 단계에서 압축·소실**

위상군 분석 결과 기존 메모리 인프라(qmd_sessions, handover_doc, TEMS, AutoMemory)는 모두 "에이전트 관점" 기록이고, 종일군의 사고 흐름을 캡처하는 유일한 공간이 노션인데 그게 파편적이라는 공백을 확인.

### 5. 종일군의 메타 결정 (위키 외부, 별도 트랙)
브레인스토밍 도중 종일군이 더 큰 결정 사항을 제기:
- **각 프로젝트마다 위상군 같은 진행자 에이전트가 필요**
- **DnT 외 프로젝트들(QuantProject 등)도 진행 파이프라인을 DnT처럼 전환**
- **기획군 서브에이전트화는 판단 실수** — 직접 소통 회복 필요
- **전문 에이전트가 자체 서브에이전트 생성 가능 (superpowers 패턴처럼)**

위상군 판단: 이 결정은 위키와 분리해야 함. 위키 스펙은 WikiAPI 인터페이스만 명시하고, 진행자 패턴은 별도 brainstorming 트랙에서 다룸. 종일군 동의 받음.

### 6. 노드 7종 확정 (핵심 고통 반영)
원래 Phase 1 MVP는 3종(Idea/Task/Lesson)이었으나, 종일군의 "테스트 디테일 소실" 고통에 대응하기 위해 **7종으로 확장**:
1. `idea` — 사용자 사고 캡처
2. `task` — 실행 단위
3. `session` — 세션 메타
4. **`test_run`** ⭐ — 테스트 디테일 원본 보존 (Fermion vs DnT 서브타입 다형성)
5. **`session_journal`** ⭐ — 마찰·실수·과정 기록
6. `lesson` — 정제된 교훈
7. `feedback` — 종일군 피드백

`test_run`의 `parameters/results/input_data`는 자유 형식 key-value (프로젝트별 다형성).

### 7. 누락 방지 3-Tier
종일군 명시: "누락 최소화가 우선, 분담 방식은 자유". 단일 기록군이 모든 책임을 지는 구조보다 **여러 층에서 같은 정보를 잡아내는 중복 안전망** 채택:
- **Tier 1**: 각 전문 에이전트가 작업 직후 `/k-capture` 직접 호출 (의무 조항)
- **Tier 2**: 기록군이 Stop hook + 명시적 호출 시 누락분 보충
- **Tier 3**: 미러 어댑터가 외부 핸드오버 자동 감지

신뢰도: Tier 1=70% / +2=90% / +3=99%+
