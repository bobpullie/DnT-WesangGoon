**상황**: Plan의 `pyproject.toml`에 패키지 경로 설정 부재. 테스트가 `from atlas.scripts.topology import ...` 로 임포트하는데, 플랫 레이아웃(`atlas/scripts/topology.py`)에선 resolve 불가.

**옵션 비교**:
- A. `pythonpath = [".."]` + Python 3.3+ implicit namespace package (채택)
- B. `pip install -e .` + `[tool.setuptools.packages.find]` — 설치 의존, 7개 엔트리포인트 모듈 미존재 → install 시 깨짐
- C. import 경로 `from scripts.topology` 변경 — plan 대규모 수정 필요

**결정 근거**: 가장 저침투적. `pyproject.toml`에 load-bearing 주석으로 보호 (`b2d23d3`).

**파일**: `pyproject.toml:25-28`

### D3. I3/I4 `k <= 1` 스킵 (Task 1.4) — **Plan 내부 모순 수정**
**상황**: Plan의 reference code (`if k == 0: continue`)가 plan의 자체 테스트와 충돌.
- `sample_l0` fixture body = 1 non-blank line
- `_make_l1(lines=200)` → L1 avg 200
- L1/L0 ratio = 200 (> 0.5 cap), L0 fan-out = 1 (< 2 cap)
- → 플랜 테스트 `test_i3_passes_when_ratio_in_range` / `test_i4_passes_with_moderate_fanout` 필패

**결정**: `if k <= 1: continue`로 변경. 근거: L0는 skeleton index root이지 prose 문서가 아니므로 L1-vs-L0 압축비/팬아웃 개념이 적용되지 않음. L0 팬아웃은 `MAX_LINES_DEFAULT[0]=200` 라인 budget (I5)으로 관리.

**의미적 정확성**: L0 자체의 구조 건강도는 I1 (유일성) + I5 (오버플로)로 커버. I3/I4는 L2 이상의 prose 계층에만 적용됨.

**파일**: `scripts/topology.py:137` (check_i3), `scripts/topology.py:161` (check_i4), 도큐멘트 docstring 업데이트 포함.

**관례로 확립**: Task 1.2~1.8 구현자 프롬프트에 "Task 1.1 convention" 으로 상기시킴.
