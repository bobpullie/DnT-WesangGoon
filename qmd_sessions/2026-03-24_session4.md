# 위상군 Session 4 — 2026-03-24
## Phase 3 Co-training 실험

**요약:** GCE Phase 3 Co-training(LoRA + GCE 공동 학습)을 0.5B/7B 모델에서 실험. 3회 반복(v1~v3) 끝에 gradient checkpointing 버그 발견. Co-training은 frozen baseline 대비 이점 없음 확인 — 모델 용량 부족(0.5B), 생성 품질 저하(7B).

**주요 성과:**
- `experiments/phase3_cotraining.py` 작성 (2-Phase: GCE warmup → LoRA+GCE co-training)
- gradient checkpointing `use_reentrant=True` 버그 발견 및 해결
- 0.5B/7B 모델에서 frozen vs co-training 비교 완료

**실험 결과:**
| Model | Phase 0 | Frozen | Co-trained |
|-------|---------|--------|------------|
| 0.5B | 60.0% | 34.2% | 25.0% |
| 7B | 86.7% | 50.0% | 34.2% |

**핵심 발견:**
1. gradient checkpointing `use_reentrant=False` 필수 (PEFT+LoRA 조합)
2. 0.5B intrinsic dimensionality 부족 → co-training 부적합
3. KL loss ↓ ≠ 생성 품질 ↑ (logit 매칭과 텍스트 품질은 별개)
4. Qwen 7B LoRA 수정 시 중국어 혼입 현상

**다음 단계:** 학습 데이터 확대, 다른 regularization 방식, 비-중국어 기반 모델 시도
