# Session 5: Kimi K2.5 모델 추가 시도 + Gemini 429 해결

## 요약
Corvis에 Kimi K2.5 (NVIDIA NIM) 추가 시도. 연결 성공했으나 응답 1~2분 지연으로 실용성 부족하여 보류 후 롤백. 과정에서 Gemini 429 에러의 근본 원인 발견 — Windows User 환경변수의 무료등급 키가 .env 결제활성화 키를 덮어쓰고 있었음. 삭제하여 해결.

## 핵심 교훈
1. **NVIDIA NIM 무료 API는 실시간 채팅에 부적합** — cold start + 공유 GPU로 1~2분 지연
2. **OpenClaw API 타입 매핑:** `openai-completions` = `/v1/chat/completions`, `openai-responses` = `/v1/responses` (OpenAI 전용)
3. **Windows env var > .env:** Docker Compose에서 시스템/유저 환경변수가 .env 파일보다 우선 (TCL #55 재확인)
4. **OpenClaw gateway token:** `down/up` 시 재생성됨. `openclaw.json`에 고정 필요
5. **docker-compose.yml environment:** 새 env var는 명시적으로 추가해야 컨테이너에 전달됨
