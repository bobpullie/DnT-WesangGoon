## 교훈/패턴
- OpenClaw gateway `mode: "lan"`은 유효하지 않음 → `bind: "0.0.0.0"` 사용 (자동으로 "lan"으로 정규화됨)
- Docker `user: root`로 시작 → apt 설치 → `su node`로 전환하여 gateway 실행 패턴
- Process 레벨 환경변수가 .env를 덮어쓸 수 있음 → `unset` 후 docker compose up
- MCP 서버의 `sys.exit(1)` 제거해야 의존 서비스(ComfyUI) 없이도 기동 가능
- MCP 서버가 `127.0.0.1`에 바인딩되면 Docker에서 접근 불가 → `0.0.0.0` 필수
